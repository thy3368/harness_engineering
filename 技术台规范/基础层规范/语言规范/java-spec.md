# Java 编码规范

## 概述

本规范基于恒生 Java 编码规范，用于指导项目 Java 开发，统一代码风格和最佳实践。

## 约束等级

| 等级 | 说明 |
|------|------|
| **强制** | 违反该项将被认为代码存在严重缺陷，全公司所有 Java 编码团队必须遵守 |
| **推荐** | 违反该项即被认为代码存在轻微缺陷，各团队根据产品特性选择性地遵守 |
| **参考** | 违反该项可被认为代码存在优化空间，各团队从持续优化角度参考使用 |

详见 [恒生 Java 编码规范](./恒生Java编码规范.md)

## 命名风格

### 类与接口

| 规则 | 说明 |
|------|------|
| **强制** | 类名使用 UpperCamelCase 风格 |
| **强制** | 以下例外：DTO/UID/DO/VO/DAO/BO/DAOImpl/YunOS/AP/PO |
| **推荐** | 抽象类命名使用 Abstract 开头 |
| **推荐** | 异常类命名使用 Exception 结尾 |
| **推荐** | 测试类以要测试的类名称开始，以 Test 结尾 |

### 方法与变量

| 规则 | 说明 |
|------|------|
| **强制** | 方法名、参数名、成员变量、局部变量使用 lowerCamelCase 风格 |
| **强制** | 常量命名全部大写，单词间用下划线隔开 |
| **强制** | 布尔类型变量不加 is 前缀，避免序列化错误 |
| **强制** | 包名统一使用小写，单数形式 |
| **强制** | 杜绝完全不规范的缩写 |

### Service 与 DAO

| 类型 | 命名规则 |
|------|----------|
| Service 接口 | `XxxService` |
| Service 实现 | `XxxServiceImpl` |
| DAO 接口 | `XxxDAO` |
| DAO 实现 | `XxxDAOImpl` |

详见 [恒生 Java 编码规范 - 命名风格](./恒生Java编码规范.md#1-命名风格)

## 代码格式

### 缩进与空格

| 规则 | 说明 |
|------|------|
| **强制** | 采用 4 个空格缩进，禁止使用 tab 字符 |
| **强制** | 一行内不能有多条语句 |
| **强制** | 方法参数在定义和传入时，多个参数逗号后边必须加空格 |
| **推荐** | 左小括号和右边的第一个字符之间不出现空格 |
| **推荐** | if/for/while/switch/do 等保留字与括号之间都必须加空格 |
| **推荐** | 任何二目、三目运算符的左右两边都需要加一个空格 |
| **推荐** | 注释的双斜线与注释内容之间有且仅有一个空格 |

### 大括号

| 规则 | 说明 |
|------|------|
| **强制** | 左大括号前不换行，左大括号后换行 |
| **强制** | 右大括号前换行，右大括号后有 else 等代码则不换行 |
| **强制** | 表示终止的右大括号后必须换行 |

### 行长度

| 规则 | 说明 |
|------|------|
| **推荐** | 单行字符数限制不超过 120 个 |
| **推荐** | 超过需要换行，遵循以下原则：第二行相对第一行缩进 4 个空格 |

详见 [恒生 Java 编码规范 - 代码格式](./恒生Java编码规范.md#3-代码格式)

## OOP 规约

### 类与对象

| 规则 | 说明 |
|------|------|
| **强制** | 避免通过类的对象引用访问静态变量或静态方法，直接用类名访问 |
| **强制** | 所有的覆写方法必须加 @Override 注解 |
| **强制** | 外部正在调用或二方库依赖的接口，不允许修改方法签名 |
| **强制** | 不能使用过时的类或方法 |
| **强制** | Object 的 equals 方法容易抛空指针异常，应使用常量或确定有值的对象来调用 equals |
| **强制** | 所有的相同类型的包装类对象之间值的比较，全部使用 equals 方法比较 |
| **强制** | String 类型之间值的比较，必须使用 equals 方法比较，禁止使用 == 或 != |
| **强制** | 所有的 POJO 类属性必须使用包装数据类型 |
| **强制** | RPC 方法的返回值和参数必须使用包装数据类型 |
| **强制** | POJO 类必须写 toString 方法 |
| **强制** | 禁止在 POJO 类中同时存在对应属性 xxx 的 isXxx() 和 getXxx() 方法 |
| **强制** | 构造方法里面禁止加入任何业务逻辑，只能用来做注入 |
| **强制** | 序列化类新增属性时，不要修改 serialVersionUID 字段 |

### 访问控制

| 规则 | 说明 |
|------|------|
| **推荐** | 类成员与方法访问控制从严 |
| **推荐** | 工具类不允许有 public 或 default 构造方法 |
| **推荐** | 类非 static 成员变量如果仅在本类使用，必须是 private |
| **推荐** | 类成员方法只供类内部调用，必须是 private |

详见 [恒生 Java 编码规范 - OOP 规约](./恒生Java编码规范.md#4-oop-规约)

## 集合处理

| 规则 | 说明 |
|------|------|
| **强制** | 只要重写 equals，就必须重写 hashCode |
| **强制** | ArrayList 的 subList 结果不可强转成 ArrayList |
| **强制** | 在 subList 场景中，高度注意对原集合元素的增加或删除 |
| **强制** | 使用集合转数组的方法，必须使用集合的 toArray(T[] array) |
| **强制** | 使用工具类 Arrays.asList() 把数组转换成集合时，不能使用其修改集合相关的方法 |
| **强制** | 不要在 foreach 循环里进行元素的 remove/add 操作 |
| **强制** | Comparator 实现类要满足三个比较条件 |
| **推荐** | 集合初始化时指定集合初始值大小 |
| **推荐** | 使用 entrySet 遍历 Map 类集合 KV |
| **推荐** | 高度注意 Map 类集合 K/V 能不能存储 null 值的情况 |

详见 [恒生 Java 编码规范 - 集合处理](./恒生Java编码规范.md#5-集合处理)

## 并发处理

| 规则 | 说明 |
|------|------|
| **强制** | 获取单例对象需要保证线程安全 |
| **强制** | 创建线程或线程池时请指定有意义的线程名称 |
| **强制** | 线程资源必须通过线程池提供，不允许在应用中自行显式创建线程 |
| **强制** | 线程池不允许使用 Executors 去创建，通过 ThreadPoolExecutor 的方式 |
| **强制** | SimpleDateFormat 是线程不安全的类，一般不要定义为 static 变量 |
| **强制** | 高并发时，同步调用应该去考量锁的性能损耗 |
| **强制** | 对多个资源、数据库表、对象同时加锁时，需要保持一致的加锁顺序 |
| **强制** | 并发修改同一记录时，避免更新丢失，需要加锁 |
| **强制** | 多线程并行处理定时任务时，使用 ScheduledExecutorService |
| **推荐** | 使用 CountDownLatch 进行异步转同步操作 |
| **推荐** | 避免 Random 实例被多线程使用 |

详见 [恒生 Java 编码规范 - 并发处理](./恒生Java编码规范.md#6-并发处理)

## 控制语句

| 规则 | 说明 |
|------|------|
| **强制** | 在 switch 块内，每个 case 要么通过 break/return 等终止，要么注释说明将继续执行到哪一个 case |
| **强制** | 在 switch 块内，必须包含一个 default 语句并且放在最后 |
| **强制** | 在 if/else/for/while/do 语句中必须使用大括号 |
| **强制** | 在高并发场景中，避免使用"等于"判断作为中断或退出的条件 |

详见 [恒生 Java 编码规范 - 控制语句](./恒生Java编码规范.md#7-控制语句)

## 异常处理

| 规则 | 说明 |
|------|------|
| **强制** | 异常不要用来做流程控制，条件控制 |
| **强制** | catch 捕获的异常必须打印异常日志 |
| **强制** | 捕获异常与抛出异常必须是完全相同的异常，或为抛出异常的父类 |
| **推荐** | 尽量使用 try-with-resources 方式 |

详见 [恒生 Java 编码规范 - 异常处理](./恒生Java编码规范.md#9-异常处理)

## 日志规约

| 规则 | 说明 |
|------|------|
| **强制** | 禁止使用 System.out 或 System.err 输出日志 |
| **强制** | 禁止在循环中打印日志 |
| **推荐** | 使用日志框架统一记录日志 |
| **推荐** | 日志级别根据环境区分：生产环境 INFO 级别，开发环境 DEBUG 级别 |

详见 [恒生 Java 编码规范 - 日志规约](./恒生Java编码规范.md#10-日志规约)

## 注释规约

| 规则 | 说明 |
|------|------|
| **强制** | 类、类属性、类方法的注释必须使用 Javadoc 规范 |
| **强制** | 方法内部单行注释，在被注释语句上方另起一行，使用 // 注释 |
| **强制** | 注释掉的代码要配合注释说明，并定期清理 |
| **推荐** | 注释要清晰表达意图，避免过度注释 |
| **推荐** | 业务逻辑代码要求加上必要的注释说明 |

详见 [恒生 Java 编码规范 - 注释规约](./恒生Java编码规范.md#8-注释规约)

## 其他规范

| 规则 | 说明 |
|------|------|
| **强制** | 禁止使用魔法值（未经预先定义的常量）直接出现在代码中 |
| **强制** | long 或 Long 赋值时，数值后使用大写的 L |
| **强制** | BigDecimal 禁止使用 double 构造函数 |
| **强制** | 禁止使用 System.out.* 或 System.err.* |
| **强制** | 禁止使用 java.sql.Date/Time/Timestamp |
| **强制** | 禁止重复 import 同一个类 |
| **强制** | 禁止 import 没用的包 |
| **强制** | 禁止在方法内对引用类型的入参赋值 |
| **推荐** | 浮点数建议使用 BigDecimal 类型 |
| **推荐** | 接口中的方法和属性不要加任何修饰符号 |

详见 [恒生 Java 编码规范 - 其它](./恒生Java编码规范.md#12-其它)

## 常量定义

| 规则 | 说明 |
|------|------|
| **推荐** | 不要使用一个常量类维护所有常量，按常量功能进行归类，分开维护 |
| **推荐** | 常量的复用层次：跨应用共享常量、应用内共享常量、子工程内共享常量、包内共享常量、类内共享常量 |
| **推荐** | 如果变量值仅在一个固定范围内变化用 enum 类型来定义 |

详见 [恒生 Java 编码规范 - 常量定义](./恒生Java编码规范.md#2-常量定义)
