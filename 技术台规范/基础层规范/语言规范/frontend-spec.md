# 前端编码规范

## 概述

本规范基于恒生前端编码规范，用于指导项目前端开发，统一代码风格和最佳实践。

详见 [恒生前端编码规范](./恒生前端编码规范.md)

## 约束等级

| 等级 | 说明 |
|------|------|
| **强制** | 违反该项将被认为代码存在严重缺陷，前端程序必须遵守 |
| **推荐** | 违反该项将被认为代码存在轻微缺陷，各团队根据产品特性选择性地遵守 |
| **参考** | 违反该项可被认为代码存在优化空间，从持续优化角度参考使用 |

详见 [恒生前端编码规范 - 约束等级定义](./恒生前端编码规范.md#2-约束等级定义)

## 环境要求

| 工具 | 推荐 |
|------|------|
| 浏览器 | Chrome（使用 Vue.js 时推荐安装 Vue.js devtools） |
| Node.js | LTS 版本，推荐使用 nvm/nvm-windows 管理多版本 |
| 编辑器 | Visual Studio Code (VS Code) |
| 格式化插件 | Prettier - Code formatter |

详见 [恒生前端编码规范 - 环境要求](./恒生前端编码规范.md#3-环境要求)

## HTML/TEMPLATE 开发规范

### 通用语法

| 规则 | 说明 |
|------|------|
| **强制** | 使用小写元素名 |
| **强制** | 使用小写属性名 |
| **强制** | 属性值使用双引号包裹 |
| **强制** | 所有元素必须有结束标签 |
| **强制** | 不要省略可选的结束标签 |
| **强制** | img 元素必须定义 alt 属性 |
| **强制** | doctype 声明使用 HTML5 标准 |

### HTML5 Doctype

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <title>页面标题</title>
  </head>
  <body></body>
</html>
```

### 语言属性

| 规则 | 说明 |
|------|------|
| **强制** | 为 html 元素设置 lang 属性 |
| **强制** | 指定字符编码为 UTF-8 |

### 引入 CSS 和 JavaScript

| 规则 | 说明 |
|------|------|
| **强制** | 引入 CSS 使用 link 标签 |
| **强制** | 引入 JavaScript 使用 script 标签 |
| **强制** | CSS 放在 head 标签内 |
| **强制** | JavaScript 放在 body 标签末尾 |
| **推荐** | 为 link 和 script 标签定义 charset 属性 |

### 属性顺序

| 顺序 | 属性类型 |
|------|----------|
| 1 | class |
| 2 | id, name |
| 3 | data-* |
| 4 | src, for, type, href |
| 5 | title, alt |
| 6 | aria-*, role |

### 语义化

| 规则 | 说明 |
|------|------|
| **强制** | 使用语义化标签代替无意义的 div |
| **推荐** | header、nav、main、section、article、aside、footer |
| **推荐** | 使用 strong 代替 b，使用 em 代替 i |

### 注释

| 规则 | 说明 |
|------|------|
| **推荐** | 注释内容首字母大写 |
| **推荐** | 注释符号与注释内容之间加一个空格 |

详见 [恒生前端编码规范 - HTML/TEMPLATE 开发规范](./恒生前端编码规范.md#41-htmltemplate-开发规范)

## CSS/LESS/SASS 开发规范

### 命名

| 规则 | 说明 |
|------|------|
| **强制** | 类名使用小写字母，以连字符分隔 |
| **强制** | id 使用驼峰命名 |
| **强制** | 避免使用标签选择器（性能问题） |
| **强制** | 不要使用元素名作为选择器（样式脆弱） |
| **推荐** | 使用有意义的类名 |
| **推荐** | 使用 BEM 命名规范 |

### 语法

| 规则 | 说明 |
|------|------|
| **强制** | 属性名冒号后面加一个空格 |
| **强制** | 每个选择器占一行 |
| **强制** | 每个属性占一行 |
| **强制** | 使用 4 个空格缩进 |
| **强制** | 属性声明后加分号 |
| **强制** | color 使用小写十六进制值 |
| **强制** | 使用 3 位十六进制颜色值（需要时使用 6 位） |
| **强制** | 省略 0 值的单位 |
| **强制** | 媒体查询放在相关样式之后 |

### 代码风格

| 规则 | 说明 |
|------|------|
| **推荐** | 使用 CSS 预处理器（LESS/SASS） |
| **推荐** | 使用 CSS 变量（CSS Custom Properties） |
| **推荐** | 使用复合属性（shorthand properties） |
| **推荐** | 使用 reset 或 normalize 重置样式 |

### 选择器权重

| 规则 | 说明 |
|------|------|
| **强制** | 避免使用 `!important` |
| **强制** | 避免使用内联样式 |
| **推荐** | 使用更低优先级的选择器 |
| **推荐** | 使用 CSS 继承 |

### 声明简写

| 属性 | 说明 |
|------|------|
| margin/padding | 使用简写：margin: 10px 20px; |
| font | 使用简写：font: 12px/1.5 sans-serif; |
| border | 使用简写：border: 1px solid #ccc; |
| background | 使用简写：background: #fff url(bg.png) no-repeat; |

### 字体规则

| 规则 | 说明 |
|------|------|
| **强制** | font-family 至少包含一个备用字体 |
| **强制** | 字体名使用引号包裹（中文或含空格时必须） |
| **推荐** | 优先使用系统字体栈 |

### 声明顺序

| 顺序 | 属性类型 |
|------|----------|
| 1 | 定位 |
| 2 | 盒模型 |
| 3 | 尺寸 |
| 4 | 文字系列 |
| 5 | 背景 |
| 6 | 其他 |
| 7 | 动画 |

详见 [恒生前端编码规范 - CSS/LESS/SASS 开发规范](./恒生前端编码规范.md#42-csslesssass-开发规范)

## JS/ES6 开发规范

### 命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 变量 | 小写字母开头，驼峰命名 | `userName`, `isValid` |
| 常量 | 全部大写，下划线分隔 | `MAX_COUNT` |
| 函数 | 小写字母开头，驼峰命名 | `getUserInfo()` |
| 类 | 大写字母开头，驼峰命名 | `class UserInfo` |
| 文件名 | 小写字母，连字符分隔 | `user-info.js` |
| 组件名 | 大写字母开头，驼峰命名 | `UserProfile.vue` |

### 语法

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `const`/`let` 代替 `var` |
| **强制** | 使用模板字符串代替字符串拼接 |
| **强制** | 使用解构赋值 |
| **强制** | 使用箭头函数 |
| **强制** | 使用严格模式 `'use strict'` |
| **强制** | 使用 `===` 代替 `==` |
| **强制** | 使用 Promises 或 async/await 代替回调函数 |
| **强制** | 使用默认参数 |
| **强制** | 使用 rest 参数代替 arguments |
| **强制** | 字符串开头和结尾不能有空格 |
| **强制** | 箭头函数参数括号和函数体括号之间有空格 |

### 代码风格

| 规则 | 说明 |
|------|------|
| **强制** | 使用 4 个空格缩进 |
| **强制** | 每行不超过 120 个字符 |
| **强制** | 多行语句使用逗号分隔（最后一项无逗号） |
| **强制** | 使用分号结尾 |
| **强制** | 对象字面量的冒号后面加空格 |
| **强制** | 数组和对象字面量最后无逗号 |
| **强制** | import/export 语句放在文件顶部 |

### 数组和对象

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `Array.isArray()` 检查数组 |
| **强制** | 使用展开运算符 `...` 复制数组/对象 |
| **强制** | 使用 `map()`/`filter()`/`reduce()` 代替循环 |
| **强制** | 避免修改数组长度 |
| **强制** | 使用 `for...of` 代替 `for...in` |

### 使用 null

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `null` 初始化已声明但未赋值的变量 |
| **强制** | 使用 `undefined` 检查未初始化变量 |
| **强制** | 不要使用 `null` 检查函数参数 |

### 文档注释

| 规则 | 说明 |
|------|------|
| **强制** | 使用 JSDoc 规范 |
| **强制** | 类必须有描述 |
| **强制** | 函数必须有描述和参数说明 |
| **强制** | 返回值必须标注类型和描述 |

```javascript
/**
 * 获取用户信息
 * @param {number} userId - 用户ID
 * @returns {Promise<UserInfo>} 用户信息
 */
function getUserInfo(userId) {
  // ...
}
```

## Vue 开发规范

### Vue 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| 组件文件 | 大写字母开头，PascalCase | `UserProfile.vue` |
| 组件名 | 大写字母开头，PascalCase | `UserProfile` |
| prop | 小写字母开头，camelCase | `userName` |
| 事件 | 小写字母开头，kebab-case | `click-info` |
| 自定义指令 | 小写字母开头，v-{name} | `v-focus` |

### 文件夹命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 组件文件夹 | 小写字母，复数形式 | `user-profile/` |
| 视图文件夹 | 小写字母，连字符 | `user-profile/` |
| 组件文件 | 大写字母开头 | `UserProfile.vue` |

### 语法

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `<script setup>` 语法 |
| **强制** | 使用 `defineProps`/`defineEmits` 声明 props/emits |
| **强制** | v-for 必须使用 key |
| **强制** | v-for 遍历时避免使用 v-if |
| **强制** | prop 必须定义类型 |
| **强制** | 组件名多个单词（组件名和目录名一致） |
| **强制** | 父子组件通信使用 props/emits |
| **强制** | 避免使用 ref 作为 reactive 对象的属性名 |
| **推荐** | 使用 Composition API |
| **推荐** | 提取可复用逻辑为 composables |

### 组件/实例的选项顺序

| 顺序 | 选项 |
|------|------|
| 1 | 公共 API（name, extends, mixins） |
| 2 | Props |
| 3 | Emits |
| 4 | setup |
| 5 | 数据（data, computed） |
| 6 | 方法（methods） |
| 7 | 生命周期钩子 |
| 8 | 渲染相关 |

### 单文件组件的顶级元素的顺序

```vue
<template>...</template>
<script setup>...</script>
<style scoped>...</style>
```

| **推荐** | 使用请求取消功能避免重复请求 |
