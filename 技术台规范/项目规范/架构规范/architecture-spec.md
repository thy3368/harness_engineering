# 项目架构规范

## 概述

本规范为项目级架构设计标准规范，参考 TOGAF 企业架构、领域驱动设计（DDD）、微服务架构模式等知名方法论，规定了架构原则、分层结构、技术选型等方面的要求。

## 架构原则

### 核心原则（基于 TOGAF）

| 原则 | 说明 | 目标 |
|------|------|------|
| **业务对齐** | 架构服务于业务目标 | 业务价值优先 |
| **松耦合** | 模块间低依赖、高内聚 | 可维护性 |
| **可演进** | 架构随业务迭代演进 | 适应变化 |
| **标准化** | 遵循统一标准和规范 | 团队协作 |
| **安全优先** | 安全贯穿架构设计 | 风险可控 |

### 架构权衡原则

| 因素 | 说明 |
|------|------|
| **性能 vs 可维护性** | 过度优化可能影响可读性 |
| **复用 vs 复杂度** | 过度抽象增加系统复杂度 |
| **敏捷 vs 规范** | 平衡速度与质量 |

### 方法论映射

| 方法论 | 应用 |
|--------|------|
| **TOGAF** | 企业级架构规划、架构开发方法 |
| **领域驱动设计 (DDD)** | 限界上下文、实体、聚合根 |
| **微服务架构** | 服务拆分、API 网关、服务治理 |
| **Clean Architecture** | 分层依赖、内聚核心 |
| **十二要素应用** | 云原生架构原则 |

详见 [架构示例](./architecture-example.md)

## 架构分层模型

### 标准分层

```
┌─────────────────────────────────────────────────────────────┐
│                      接入层 (Gateway)                         │
│          API Gateway / Load Balancer / CDN                  │
├─────────────────────────────────────────────────────────────┤
│                      展示层 (Presentation)                    │
│           Web / Mobile / API / Events                       │
├─────────────────────────────────────────────────────────────┤
│                      应用层 (Application)                    │
│              Use Cases / Application Services                │
├─────────────────────────────────────────────────────────────┤
│                      领域层 (Domain)                          │
│         Entities / Value Objects / Domain Services          │
├─────────────────────────────────────────────────────────────┤
│                   基础设施层 (Infrastructure)                │
│       Repositories / External Services / Messaging           │
└─────────────────────────────────────────────────────────────┘
```

### 分层职责

| 层级 | 职责 | 依赖方向 |
|------|------|----------|
| **接入层** | 请求路由、协议转换、限流熔断 | 无 |
| **展示层** | 用户交互、数据展示、协议响应 | 依赖应用层 |
| **应用层** | 用例编排、事务管理、事件发布 | 依赖领域层 |
| **领域层** | 业务规则、领域模型、领域事件 | 无外部依赖 |
| **基础设施层** | 持久化、外部服务、消息发送 | 被各层依赖 |

### 依赖规则

> **核心原则**：内层不依赖外层，外层依赖内层；依赖抽象而非具体实现。

```java
// 错误：领域层依赖基础设施
class OrderService {
    @Autowired
    private JdbcOrderRepository repository;  // ❌ 违反依赖规则
}

// 正确：通过接口抽象
class OrderService {
    @Autowired
    private OrderRepository repository;  // ✅ 依赖抽象
}
```

## 领域驱动设计规范

### 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| **限界上下文** | 业务边界 | 订单上下文、支付上下文 |
| **聚合根** | 领域实体根 | Order、Payment |
| **实体** | 有唯一标识 | OrderItem |
| **值对象** | 无标识、不可变 | Money、Address |
| **领域服务** | 跨实体业务逻辑 | PricingService |
| **领域事件** | 领域内事件 | OrderCreatedEvent |

### 聚合设计原则

| 原则 | 说明 |
|------|------|
| **聚合内强一致性** | 聚合内数据保持一致 |
| **聚合间最终一致性** | 跨聚合通过事件最终一致 |
| **聚合根负责** | 外部只能通过聚合根访问 |
| **小聚合** | 聚合边界应尽量小 |

### 领域模型示例

```java
// 聚合根
public class Order {
    private OrderId id;
    private List<OrderItem> items;
    private Money totalAmount;
    
    // 只能通过聚合根方法修改内部状态
    public void addItem(OrderItem item) {
        // 业务规则校验
        validateItem(item);
        items.add(item);
        recalculateTotal();
    }
}
```

## 微服务架构规范

### 服务拆分原则

| 原则 | 说明 |
|------|------|
| **业务边界** | 按业务领域拆分 |
| **单一职责** | 每个服务只做一件事 |
| **高内聚低耦合** | 服务内聚、服务间松耦合 |
| **独立性** | 可独立部署、升级 |

### 服务通信

| 通信方式 | 使用场景 | 示例 |
|----------|----------|------|
| **同步调用 (REST/gRPC)** | 实时交互 | 查询订单 |
| **异步消息** | 耗时操作、事件驱动 | 订单创建后发送通知 |
| **事件驱动** | 跨服务状态同步 | 订单状态变更 |

### 服务治理

| 组件 | 说明 | 推荐技术 |
|------|------|----------|
| **服务注册/发现** | 动态服务发现 | Nacos、Consul |
| **配置中心** | 集中配置管理 | Nacos、Apollo |
| **API 网关** | 统一入口、限流、鉴权 | Kong、Spring Cloud Gateway |
| **熔断降级** | 故障隔离 | Sentinel、Hystrix |
| **链路追踪** | 分布式追踪 | SkyWalking、Jaeger |
| **服务监控** | 指标收集 | Prometheus、Grafana |

## 架构质量属性

### 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| **TPS** | 根据业务确定 | 核心接口 ≥50，高频 ≥100 |
| **响应时间 RT** | < 2s | 对外接口响应时间 |
| **启动时间** | < 150s | 微服务启动时长 |
| **并发能力** | 根据峰值设计 | 支持业务峰值 |

### 可靠性要求

| 指标 | 要求 | 说明 |
|------|------|------|
| **可用性** | ≥ 99.9% | 年度停机时间 < 8.76 小时 |
| **RTO** | ≤ 30min | 恢复时间目标 |
| **RPO** | ≤ 20min | 恢复点目标 |
| **容错** | 无单点故障 | 数据库、缓存、应用的 HA |

### 安全性要求

| 能力 | 说明 |
|------|------|
| **身份认证** | 统一身份认证、细粒度权限控制 |
| **传输安全** | HTTPS/SSL 加密传输 |
| **存储安全** | 敏感数据加密存储 |
| **展示脱敏** | 敏感信息展示脱敏 |
| **安全审计** | 操作日志记录，保存 ≥ 3 个月 |
| **漏洞防护** | CSRF、XSS、XXE、SQL 注入防护 |

### 可维护性要求

| 能力 | 说明 |
|------|------|
| **日志管理** | 区分系统日志/业务日志、链路跟踪 |
| **监控能力** | 运行状态监控、业务可用性监控 |
| **问题定位** | APM 工具支持 |
| **配置管理** | 外部化配置、环境隔离 |

| 维度 | 权重 | 评估项 |
