# 约束条件规格

## 概述

本文档详细说明MC3.0在各类场景下的约束条件和限制，用于帮助开发者和运维人员理解系统边界。

---

## 一、消息可靠性约束

### 1.1 内存级可靠等级（RPO非0）

当消息可靠等级配置为内存级时：

| 断开场景 | 影响 | 说明 |
|----------|------|------|
| 生产者与主分区断开 | 消息丢失 | 连接断开期间的消息直接丢弃 |
| 消费者与会话断开 | 消息丢失 | 会话断开期间的消息直接丢弃 |
| 主备分区网络断开 | 消息丢失 | 集群间同步的消息丢弃 |

**结论**：内存级模式下，连接断开会导致消息丢失，不适合对可靠性要求高的场景。

### 1.2 文件级可靠等级（RPO非0）

当消息可靠等级配置为文件级时：

#### 场景1：生产者崩溃

- **影响**：切换到新机器后，原生产者未确认的消息丢失
- **原因**：本地持久化的未确认消息无法恢复

#### 场景2：主分区崩溃，备分区切为主

可能发生消息丢失的3种情况：

| 情况 | 原因 | 结果 |
|------|------|------|
| 单生产者+补缺被覆盖 | 补缺消息被新发布消息覆盖 | 补缺失败，消息丢失 |
| 多生产者+时序不一致 | 补缺重排时序与原主分区不一致 | 可能丢失消息 |
| ACK未同步备分区 | 主分区回复ACK但未同步到备分区 | 补缺无法触发，消息丢失 |

### 1.3 发布到mc2.0场景

| 场景 | 影响 |
|------|------|
| mc节点主备切换 | 丢弃重试消息（通常32个消息） |
| t2sdk发布失败 | 记录日志并丢弃，无重试 |

## 二、资源限制

### 2.1 磁盘空间限制

#### 生产者磁盘空间不足

| 影响 | 说明 |
|------|------|
| 发布缓存文件创建失败 | 发布接口返回失败 |
| mcbiz异步发布场景 | 继续重试，可能导致异步队列溢出 |
| 再次启动清理失败 | 使用老的发布缓存 |

#### MC节点磁盘空间不足

| 影响 | 说明 |
|------|------|
| 消息文件创建失败 | 返回发布失败，由mcsdk自动重试 |
| 索引文件生成失败 | 补缺订阅一直失败 |
| 集群偏移量写失败 | 消费偏移量保存失败 |

#### 消费者磁盘空间不足

| 影响 | 说明 |
|------|------|
| 消费偏移量文件创建失败 | 再次订阅后可能重复消费历史消息 |

### 2.2 消息池限制

#### 生产者消息池不足

| 场景 | 影响 |
|------|------|
| 异步发布获取MsgWriter失败 | 发布失败，由调用者重试 |
| 同步发布获取MsgReader失败 | 返回发布超时错误 |
| 同步发布获取MsgWriter失败 | 发布失败 |

#### MC节点消息池不足

| 场景 | 影响 |
|------|------|
| 处理发布获取MsgReader失败 | 丢失发布给mc2.0的消息，无重试 |
| 处理会话断开获取MsgWriter失败 | 断开失败，需等重试次数到了再断开 |
| 处理消息推送获取MsgWriter失败 | 断开会话，取消该会话上的订阅项 |

#### 消费者消息池不足

| 场景 | 影响 |
|------|------|
| 提交消费偏移量获取MsgWriter失败 | 提交失败，偏移量未保存 |

### 2.3 内存限制

| 场景 | 说明 |
|------|------|
| 服务端内存不足 | 类似消息池不足的处理方式，获取消息失败导致服务异常 |

### 2.4 数据库版本

数据库版本要求请参考：https://iknow.hs.net/portal/docView/sharing/120069

## 三、集群扩展限制

### 3.1 消费者侧限制

#### 地址配置要求

| 要求 | 说明 |
|------|------|
| 必须配置全部节点 | mc集群地址不能只填部分，需配置全部mc集群节点 |
| 节点间分隔 | 多个节点用逗号分隔 |
| 扩展后需更新 | mc集群扩展时，消费者集群地址需更新并补全，然后重启 |

#### 不按要求配置的后果

| 场景 | 影响 |
|------|------|
| 新增节点有主分区 | 消费者订阅会一直失败 |
| 除新增节点外其他节点停止 | 消费者不能和该节点创建会话和订阅消费 |

### 3.2 运行时限制

| 限制 | 说明 |
|------|------|
| 不支持运行时清理持久化 | mcsdk订阅或发布进程存在时，mc服务端停止和清理持久化信息暂时不支持 |
| 可能导致的异常 | 消费序号或发布序号异常 |

## 四、约束速查表

### 4.1 可靠性与约束对照

| 可靠等级 | 网络断开 | 生产者崩溃 | 主备切换 | 磁盘不足 |
|----------|----------|------------|----------|----------|
| 内存级 | 丢弃 | 丢弃 | 丢弃 | 缓存失败 |
| 文件级 | 丢弃 | 丢失 | 可能丢失 | 写入失败 |

### 4.2 资源不足影响矩阵

| 资源类型 | 生产者 | MC节点 | 消费者 |
|----------|--------|--------|---------|
| 磁盘不足 | 发布失败 | 消息/索引失败 | 偏移量失败 |
| 消息池不足 | 发布失败/超时 | 消息丢失/断开 | 偏移量失败 |
| 内存不足 | - | 服务异常 | - |

### 4.3 扩展约束检查清单

- [ ] mc集群地址是否配置了全部节点
- [ ] 多个节点是否用逗号分隔
- [ ] 扩展后是否更新了消费者地址
- [ ] 扩展后是否重启了消费者
- [ ] mcsdk运行时是否避免清理持久化
