# 实体定义规格

## 概述

本文档详细定义消息中心3.0（MC3.0）的核心实体及其属性，用于研发人员理解数据模型和接口设计。

---

## 一、接口实体

### 1.1 适配接口

| 实体 | 说明 | 预定义 | 必填 |
|------|------|--------|------|
| 适配接口 | MC3.0适配不同客户端的接口 | 否 | 是 |

#### 支持的客户端SDK

| SDK类型 | 说明 |
|--------|------|
| kafka-sdk | Kafka协议适配 |
| mqapi | MQ协议适配 |
| mqsdk | MC SDK适配 |
| wss | WebSocket安全协议适配 |

### 1.2 发布订阅接口

| 实体 | 说明 | 预定义 | 必填 |
|------|------|--------|------|
| 发布接口 | 生产者的发布接口 | 否 | 是 |
| 订阅接口 | 消费者的订阅接口 | 否 | 是 |

### 1.3 管理接口

| 实体 | 说明 | 预定义 | 必填 |
|------|------|--------|------|
| 管理接口 | 调用MC3.0所有管理功能的接口 | 否 | 是 |

## 四、主题实体

### 4.1 主题核心属性

| 属性 | 类型 | 说明 | 必填 |
|------|------|------|------|
| 主题名 | String | 同一主题下有多个分区，这些分区的主题名相同 | 是 |
| 队列 | String | 通过队列名去区分死信队列、重试队列以及定时队列 | 是 |
| 分区 | String | 分区描述如"分区"实体描述 | 是 |
| 过滤条件 | String | 通过设置过滤条件（16个）来进行消息过滤，并支持集合订阅 | 否 |
| 可靠等级 | Enum | 支持主题的配置文件中配置主题为内存级（至多一次）或文件级（至少一次） | 是 |
| 消息有效期 | Enum | 支持消息有效期管理，删除已过期失效消息，释放资源 | 是 |
| 顺序消息 | Enum | 包含全局有序、分区有序以及无序消费 | 是 |
| 强一致 | Enum | 包含集群RPO以及机房RPO | 是 |

### 4.2 消息有效期等级

| 等级 | 有效期 |
|------|--------|
| 0 | 一天 |
| 1 | 二天 |
| 2 | 三天 |
| 3 | 一星期 |
| 4 | 二星期 |
| 5 | 一个月 |
| 6 | 三个月 |
| 7 | 六个月 |
| 8 | 一年 |

### 4.3 顺序消息类型

| 类型 | 说明 |
|------|------|
| 全局有序 | 所有消息按发送顺序消费 |
| 分区有序 | 同一分区内的消息按发送顺序消费 |
| 无序消费 | 不保证消息顺序 |

### 4.4 强一致类型

| 类型 | 说明 |
|------|------|
| 集群RPO | 保证单个集群内的消息不丢失 |
| 机房RPO | 保证跨机房的消息不丢失 |

### 4.5 主题消息属性

| 属性 | 说明 | 必填 |
|------|------|------|
| 主题分区 | 该消息所属的主题分区，可使用主题分区编号或主题名+分区号标识 | 是 |
| 消息序号 | 由发布者为该消息生成的序号 | 是 |

| 属性 | 说明 | 必填 |
|------|------|------|

| 属性 | 说明 | 必填 |

| 通讯端口 | 该主题分区的发布端与订阅端通讯使用的端口 | 是 |

## 七、订阅关系实体

### 7.1 消费模式

| 模式 | 说明 |
|------|------|
| 广播消费 | 任意一条消息都会被集群内所有消费者消费 |
| 集群消费 | 任意一条消息只需要被集群内的任意一个消费者处理即可，同一个消费组下的各个消费端，会使用负载均衡的方式进行消费 |

### 7.2 订阅关系属性

| 属性 | 说明 |
|------|------|
| 订阅关系 | 包含广播消费模式以及集群消费模式 |

---

## 八、消息存储实体

### 8.1 消息落地

| 实体 | 说明 |
|------|------|
| 事物状态表 | 针对事物消息为preparedType和rollbackType的存储的是事物状态表的索引偏移量 |

### 8.2 消息索引

| 实体 | 说明 |
|------|------|
| 订阅索引项 | 消费者所订阅消息的索引项 |
| 消息索引文件 | 持久化分区中所有消息的索引文件 |
| 特征值索引文件 | 持久化分区中所有消息特征值的索引文件 |
| 时间索引文件 | 持久化分区中所有消息时间戳的索引文件 |

### 8.3 消费偏移

| 实体 | 说明 |
|------|------|
| 集群消费偏移文件 | 记录集群消费模式下消费者的消费进度 |
| 广播消费偏移文件 | 记录广播消费模式下消费者的消费进度 |

---

## 九、版本与缓存实体

### 9.1 MC版本

| 属性 | 说明 |
|------|------|
| McVersion | 每次mc服务端被清理后，由主节点产生的mc版本号，用以保证生产者和消费者的序号的一致性 |

### 9.2 发布缓存

| 实体 | 说明 |
|------|------|
| 本地缓存文件 | 客户端的发布缓存，当收到确认后才会将缓存中的消息进行清理 |

## 十一、实体属性速查

| 主题 | 主题名、队列、分区、可靠等级、消息有效期、顺序消息 |

| 属性 | 可选值 |
|------|--------|
| 可靠等级 | 内存级、文件级 |
| 消息有效期 | 一天、二天、三天、一星期、二星期、一个月、三个月、六个月、一年 |
| 顺序消息 | 全局有序、分区有序、无序消费 |
| 强一致 | 集群RPO、机房RPO |
| 消费模式 | 广播消费、集群消费 |
