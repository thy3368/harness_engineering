# 业务功能树规范

## 1 规范说明

业务功能树是用于描述业务系统功能架构的标准化文档，采用层级结构组织，从宏观到微观逐级细化业务功能。本规范定义了业务功能树的组织结构、层级定义、字段说明及编写要求。

### 1.1 规范目的

- 统一业务功能描述方式，便于各团队沟通与理解
- 为需求分析、系统设计、测试用例编写提供功能基准参考
- 支持功能映射与追溯，关联设计文档、接口规范、DFX要求
- 沉淀业务知识，形成可维护的功能规范文档

### 1.2 适用范围

本规范适用于消息中心3.0系统及其他分布式中间件业务功能树的编写，包括但不限于消息队列、配置中心、注册中心等系统。

---

## 2 层级定义

业务功能树采用六级层级结构，自上而下依次为：业务领域、价值流、活动、任务、关键字段。

| 层级 | 名称 | 定义说明 | 示例 |
|:---:|:---:|:---|:---|
| 一级 | 业务领域 | 业务系统或独立服务边界 | 消息中心3.0 |
| 二级 | 价值流 | 业务价值导向的功能分组 | ldp_mcsdk、ldp_mc |
| 三级 | 活动 | 面向用户或系统的核心业务流程 | 发布、订阅、管理 |
| 四级 | 任务 | 具体的业务操作单元 | 创建配置对象、初始化生产者、订阅消息 |
| 五级 | 子任务 | 任务的细化步骤（可选） | 设置目标mc地址、设置生产者集群名 |
| 六级 | 关键字段 | 任务涉及的配置或数据项 | server_address、group_name、msg_count |

## 3 字段说明

业务功能树表格中各字段的定义及填写要求如下：

| 字段名称 | 类型 | 必填 | 说明 | 示例 |
|:---|:---:|:---:|:---|:---|
| 业务领域 | 字符串 | 是 | 一级功能分类，对应业务系统名称 | 消息中心3.0 |
| 价值流 | 字符串 | 是 | 二级功能分组，体现业务价值导向 | ldp_mcsdk、ldp_mc |
| 活动 | 字符串 | 是 | 三级功能模块，面向用户或系统的业务流程 | 发布、订阅、管理 |
| 任务 | 字符串 | 是 | 四级功能点，具体的业务操作单元 | 创建配置对象、创建生产者 |
| 子任务 | 字符串 | 否 | 五级功能点，任务的细化步骤 | 设置目标mc地址、绑定配置对象 |
| 关键字段 | 字符串 | 否 | 六级配置项，任务涉及的配置或数据项名称 | server_address、group_name |
| 描述 | 字符串 | 是 | 功能点的详细说明，阐明功能用途和行为 | 该配置对象，主要用于设置生产者相关配置 |
| 备注 | 字符串 | 否 | 补充说明，包括限制条件、前置依赖、使用注意等 | 判断重名的前提是在同一个工作路径下 |
| DFX | 字符串 | 否 | 性能、可靠性、兼容性、安全性等非功能要求 | 消息吞吐能力：性能总数达到5W tps |
| 接口 | 字符串 | 否 | 对应的内部或外部接口编号及名称 | 620924（内部接口） |
| 设计文档 | 字符串 | 否 | 相关设计文档的链接地址 | MC3.0一期概要设计.pdf |
| 修改说明 | 字符串 | 否 | 功能点的变更记录 | 新增、修改、移除 |
| 修改版本 | 字符串 | 否 | 变更对应的效能版本号 | LDP1.0V202401.07.000 |
| 关联需求 | 字符串 | 否 | 关联的需求编号 | #202412064383 |

## 4 编写要求

### 4.1 命名规范

- 任务名称应使用动宾短语，简洁明了
- 关键字段应与代码实现中的变量名保持一致
- 层级名称应使用业务术语，避免技术实现细节

### 4.2 描述要求

- 描述应阐明功能用途、核心行为及与其他任务的关联
- 涉及限制条件时，应明确说明约束范围和影响
- 涉及多个步骤时，应按业务流程顺序描述

### 4.3 关联要求

- 每个功能点应关联对应的设计文档
- 关键接口应注明接口编号和用途
- DFX要求应量化描述，便于验证

## 5 示例模板

### 5.1 YAML格式（推荐）

YAML格式兼顾可读性、可解析性和版本控制diff清晰度，适用于文档编写、程序处理和团队协作场景。

```yaml
# 业务功能树示例 - 消息中心3.0 发布模块
business_domain: 消息中心3.0
value_streams:
  - name: ldp_mcsdk
    description: 消息中心SDK，提供客户端消息发布订阅能力
    activities:
      - name: 发布
        description: 生产者消息发布流程
        tasks:
          - name: 创建配置对象
            key_field: NewEmptyConfig
            description: 该配置对象，主要用于设置生产者相关配置。多进程情况下，创建对象的名字均不一样
            notes: 调用生产者管理功能-查看客户端配置信息
            sub_tasks:
              - name: 设置目标mc地址
                key_field: server_address
                description: mc地址对应服务端broker.json对应的地址，如果有多个用逗号分割
                constraints:
                  - 详见DFX-系统容量-服务集群成员个数
              - name: 设置生产者集群名
                key_field: group_name
                description: 生产者集群的名字，加上mac地址和随机数，生成生产者的唯一名字
                constraints:
                  - 发布者名最大32字节，唯一名字最大58字节
                  - 集群名中不支持 / ? @ $ & ( ) \ | ' ' " " : 等符号
                  - Windows端禁止使用中文

          - name: 创建生产者
            key_field: NewProducer
            description: 该生产者对象，主要用于提供发布相关功能
            notes: 调用生产者管理功能-获取生产者列表

          - name: 同步/异步发布
            tasks:
              - name: 创建消息对象
                key_fields:
                  - 主题名
                  - 过滤条件
                  - 业务体
                description: 构建待发布的消息，包含主题、过滤条件和业务数据
                constraints:
                  - 主题名最大256字节
                  - 过滤条件最多16个，每个最大63字节
                  - 业务体单行最大16MB

              - name: 消息发布
                key_field: PublishMsg
                description: 先写入发送队列，再尝试写入发布窗口进行发布
                dfx:
                  performance: 消息吞吐能力5W tps，单主题不少于3W
                  reliability: RTO主备切换5s，TCP连接切换90s
                interface: 620924

      - name: 订阅
        # ... 类似结构

      - name: 管理
        # ... 类似结构
```

### 6.2 关键字段类型说明

| 字段类型 | 说明 | 示例 |
|:---|:---|:---|
| 配置类 | 系统运行时配置参数 | server_address、group_name、msg_count |
| 状态类 | 运行时状态标识 | 事件类型、处理结果、返回方式 |
| 数据类 | 业务数据载体 | 主题名、过滤条件、业务体 |
| 序号类 | 消息流转标识 | 订阅索引号、主题分区偏移量 |
