# 架构规范

## 架构目标

- 支持投资端全业务的资金划转，提供电子直连、邮件、传真等多种划款方式
- 支持根据银行间下行接口自动生成指令，并能够根据结算数据自动交收
- 支持TA文件、估值文件的自动导入，生成对应TA、费用业务指令
- 支持产品账户等业务数据的跨平台同步以及批量导入
- 提供划款单自动生成、成交单附件自动盖章等功能

## 架构要求

### 性能要求

- **强制** 核心查询接口：TPS ≥ 50，响应时间 RT < 2s
- **强制** 高频查询接口：TPS ≥ 100，响应时间 RT < 1s
- **强制** 服务启动时间：微服务启动时长 < 150秒
- **强制** 对外接口响应时间：RT < 2s
- **推荐** 采用缓存、异步处理等优化手段提升系统性能

### 可靠性要求

- **强制** 系统可用性：服务水平 ≥ 99.9%
- **强制** 数据可用性：日志类数据保存3个月
- **强制** 高可用部署：数据库、缓存、应用服务、反向代理均支持高可用部署（主备、集群），无单点故障
- **强制** 故障恢复：RTO ≤ 30min，RPO ≤ 20min
- **强制** 容错能力：支撑系统运行的数据库、缓存等发生单台故障时，系统仍可提供服务
- **推荐** 支持同城灾备+异地灾备等多种部署架构

### 安全性要求

- **强制** 漏洞防护：具备CSRF、XSS、XXE、SQL注入等防护能力
- **强制** 身份认证：使用OMC2.0统一身份认证中心，实施细粒度权限控制
- **强制** 敏感数据保护：传输加密（HTTPS/SSL）、存储加密、展示脱敏
- **强制** 访问控制：实施IP白名单机制，对所有接口进行访问控制
- **强制** 安全审计：记录系统操作日志，保存时间不少于3个月

### 可维护性要求

- **强制** 日志管理：完善的运行日志和业务日志，区分系统日志，具有链路跟踪标识traceId，日志级别区分，默认INFO及以上
- **强制** 监控能力：提供节点内部运行状态监控接口（线程、队列等），提供业务可用性监控接口，支持SEE平台集成监控
- **推荐** 使用APM工具进行性能问题定位和分析

### 运行环境兼容性

- **强制** 支持非信创环境：CentOS 7.5+/Redhat 7.9+、Oracle 11g+/MySQL 5.7+、JDK 1.8+
- **强制** 支持信创环境：麒麟V10、达梦V8/OceanBase 3.2.X+/4.2.X、东方通TongWeb/宝兰德BES、龙井JDK 8.14.15+

## 架构分层

### 展示/对接层

- **前端架构**：基于HUI2.0框架开发，支持前后端分离
- **外部对接**：通过传真、邮件、支付等网关与托管行进行对接

### 网关层

- **负载均衡**：使用HSIAR提供流量分发的负载均衡服务
- **网关服务**：传真网关、邮件网关、支付网关等，提供对接托管行的能力
- **流量控制**：实施限流、熔断、降级等措施，保障系统稳定性

### 业务服务层

- **核心服务**：以COP-WEB单体应用形式承载，包含指令、勾销、对账、债券结算等核心业务逻辑
- **服务拆分**：遵循业务领域边界，逐步推进微服务化改造
- **业务组件**：按功能模块划分业务组件，提高代码复用性

### 数据存储层

- **数据库支持**：Oracle/MySQL/达梦/OceanBase
- **缓存策略**：使用Redis缓存热点数据，提高数据访问效率
- **文件存储**：统一的文件存储管理，支持文件的上传、下载、预览等功能

### 支撑服务层

- **操作员中心**：OMC2.0，提供统一的用户管理、认证授权服务
- **任务调度**：支持定时任务、异步任务的管理和执行
- **工作流系统**：支持业务流程的定义、执行和监控
- **服务治理**：支持服务注册发现、配置管理、服务监控等
- **消息队列**：RabbitMQ，支持异步通信和削峰填谷
- **分布式协调**：ZooKeeper，提供分布式锁、配置管理等功能
