# C/C++ 编码规范

## 概述

本规范基于恒生 C/C++ 编码规范，用于指导项目 C/C++ 开发，统一代码风格和最佳实践。

## 约束等级

| 等级 | 说明 |
|------|------|
| **强制** | 必须做的项，违反该项将被认为代码存在严重缺陷，全公司所有 C/C++ 编码团队必须遵守 |
| **推荐** | 违反该项即被认为代码存在轻微缺陷，各团队根据产品特性选择性地遵守 |
| **参考** | 违反该项可被认为代码存在优化空间，各团队从持续优化角度参考使用 |

详见 [恒生 C、C++ 编码规范](./恒生C、C++编码规范.md)

## 布局结构规范

### 一般规则

| 规则 | 说明 |
|------|------|
| **强制** | 每个源程序文件的头部必须包含文件头部说明和修改记录 |
| **强制** | 每个函数头部必须包含函数头部说明，内部函数头部至少需要描述函数的主要功能 |
| **强制** | 每个源程序文件的尾部增加一行空行 |

### 文件头部说明

| 规则 | 说明 |
|------|------|
| **强制** | 源程序的文件头部说明必须包含：系统名称、模块名称、文件名称、软件版权、功能说明、系统版本、开发人员、开发时间和修改记录 |

### 函数头部说明

| 规则 | 说明 |
|------|------|
| **强制** | 函数头部说明必须包含：函数功能、输入参数、输出参数、返回值、调用关系 |
| **强制** | 内部函数头部至少需要描述函数的主要功能 |

### 头文件结构

```c
#ifndef _MODULE_FILE_H_
#define _MODULE_FILE_H_

// 头文件内容

#endif
```

### 源文件结构

| 规则 | 说明 |
|------|------|
| **强制** | 源文件结构顺序：文件头部说明、include 部分、宏定义、类型定义、函数原型、全局变量定义、函数实现 |

详见 [恒生 C、C++ 编码规范 - 布局结构规范](./恒生C、C++编码规范.md#三-布局结构规范)

## 书写排版规范

### 一般规则

| 规则 | 说明 |
|------|------|
| **强制** | 采用 4 个空格缩进，禁止使用 tab 字符 |
| **强制** | 一行代码长度不超过 120 个字符 |
| **强制** | 关键字与其后的左括号之间必须有一个空格 |
| **强制** | 函数名与左括号之间不出现空格 |
| **强制** | 赋值运算符、逻辑运算符、关系运算符前后必须加空格 |

### 空格规范

| 规则 | 说明 |
|------|------|
| **强制** | `if`、`while`、`for`、`switch` 等关键字与左括号之间加空格 |
| **强制** | 逗号后面加空格，分号后面不加空格（在同一行时） |
| **强制** | 二目、三目运算符前后加空格 |
| **推荐** | `sizeof` 关键字与左括号之间不加空格 |

### 空行规范

| 规则 | 说明 |
|------|------|
| **强制** | 在不同逻辑部分的代码之间插入空行 |
| **强制** | 函数之间插入空行 |
| **强制** | 头文件保护 `#ifndef` 与第一个 `#endif` 之间插入空行 |
| **推荐** | 连续的变量声明之间不插入空行 |

### 对齐规范

| 规则 | 说明 |
|------|------|
| **推荐** | 续行的缩进与同一结构层次的第一行对齐 |
| **推荐** | 函数参数与第一个参数对齐 |
| **推荐** | 结构成员赋值时，赋值符号对齐 |

详见 [恒生 C、C++ 编码规范 - 书写排版规范](./恒生C、C++编码规范.md#四-书写排版规范)

## 命名规范

### 一般规则

| 规则 | 说明 |
|------|------|
| **强制** | 标识符只能由英文字母、数字和下划线组成，且不能以数字开头 |
| **强制** | 不能使用中文拼音或纯拼音作为标识符 |
| **强制** | 禁止使用系统中已有的保留标识符 |

### 标示符命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 宏 | 全部大写，以下划线分隔 | `MAX_BUFFER_SIZE` |
| 枚举常量 | 同宏命名规则 | `ENUM_VALUE_ONE` |
| 类名 | 大写字母开头，驼峰命名 | `class UserInfo` |
| 函数名 | 小写字母开头，驼峰命名 | `getUserInfo()` |
| 变量名 | 小写字母开头，驼峰命名 | `userInfo`, `pUserInfo` |
| 成员变量 | 以 `m_` 或 `m` 开头 | `m_userInfo` |
| 常量 | 全部大写，以下划线分隔 | `MAX_COUNT` |
| 文件名 | 全部小写，以下划线分隔 | `user_info.cpp` |

### 文件命名

| 类型 | 规则 |
|------|------|
| 头文件 | `.h` 扩展名 |
| 源文件 | `.cpp` 或 `.c` 扩展名 |
| 实现文件 | `.inl` 扩展名（内联函数） |

详见 [恒生 C、C++ 编码规范 - 命名规范](./恒生C、C++编码规范.md#五-命名规范)

## 语言规范

### 常量

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `const` 代替 `#define` 定义常量 |
| **强制** | 使用 `enum` 代替 `#define` 定义枚举常量 |
| **强制** | 整型常量应加上后缀标识类型：`L` 表示 long，`LL` 表示 long long |
| **推荐** | 浮点常量加上小数点标识类型 |

### 变量

| 规则 | 说明 |
|------|------|
| **强制** | 变量在定义时必须初始化 |
| **强制** | 禁止使用未初始化的变量 |
| **强制** | 指针变量应初始化为 `NULL` |
| **强制** | 循环体内不应定义变量 |
| **推荐** | 局部变量尽量缩小作用域 |
| **推荐** | 使用有意义的变量名 |

### 预处理

| 规则 | 说明 |
|------|------|
| **强制** | 头文件必须使用 `#ifndef` 进行保护 |
| **强制** | `#include` 使用绝对路径或项目相对路径 |
| **强制** | 避免多层嵌套的 `#include` |
| **推荐** | 系统头文件放在项目头文件之前 |

### 宏

| 规则 | 说明 |
|------|------|
| **强制** | 宏定义必须使用括号保护表达式 |
| **强制** | 带参数的宏定义必须使用 `do { } while(0)` 结构 |
| **强制** | 避免使用带副作用的宏参数 |
| **推荐** | 尽量使用 `inline` 函数代替宏 |

### 表达式

| 规则 | 说明 |
|------|------|
| **强制** | 避免隐式类型转换 |
| **强制** | 避免使用逗号表达式 |
| **强制** | 避免使用 `goto` 语句 |
| **强制** | 表达式中相同类型的变量不应出现多次计算 |
| **推荐** | 使用括号明确表达式的优先级 |

### 语句

| 规则 | 说明 |
|------|------|
| **强制** | `if`、`while`、`for`、`do` 语句必须使用大括号 |
| **强制** | `switch` 语句必须包含 `default` 分支 |
| **强制** | `case` 语句必须使用 `break` 终止 |
| **推荐** | 使用 `for` 循环代替 `while` 循环（当循环变量为整数时） |
| **推荐** | 避免多层嵌套的 `if` 语句 |

### 注释

| 规则 | 说明 |
|------|------|
| **强制** | 文件头部必须有注释说明 |
| **强制** | 函数头部必须有注释说明 |
| **强制** | 注释必须与代码保持一致 |
| **强制** | 修改代码必须同步修改注释 |
| **强制** | 不允许注释掉的代码存在 |
| **推荐** | 使用 `//` 进行行注释，使用 `/* */` 进行块注释 |

### 类

| 规则 | 说明 |
|------|------|
| **强制** | 类的成员变量必须在构造函数或初始化列表中初始化 |
| **强制** | 拷贝构造函数和拷贝赋值运算符必须成对出现或显式删除 |
| **强制** | 基类的析构函数必须为 virtual |
| **强制** | 禁止使用裸指针，使用智能指针 |
| **推荐** | 使用 `override` 关键字标记覆盖函数 |
| **推荐** | 使用 `explicit` 关键字标记单参数构造函数 |
| **推荐** | 成员函数尽量使用 `const` 修饰 |

### 类型转换

| 规则 | 说明 |
|------|------|
| **强制** | 避免使用 C 风格类型转换，使用 `static_cast`、`reinterpret_cast` |
| **强制** | 使用 `dynamic_cast` 进行向下转型 |
| **强制** | 使用 `const_cast` 去除 const 属性 |
| **推荐** | 使用 `static_cast` 进行显式类型转换 |

### 指针和数组

| 规则 | 说明 |
|------|------|
| **强制** | 指针变量应与解引用运算符对齐 |
| **强制** | 使用 `sizeof(变量名)` 代替 `sizeof(类型)` |
| **强制** | 数组定义时指定大小，使用 `sizeof` 计算数组元素个数 |
| **推荐** | 使用 `std::array` 或 `std::vector` 代替原始数组 |

### 异常

| 规则 | 说明 |
|------|------|
| **强制** | 异常必须使用 `try-catch` 捕获 |
| **强制** | 捕获的异常必须进行处理 |
| **强制** | 不允许抛出指针类型的异常 |
| **推荐** | 使用标准异常类型 |

| **强制** | 代码提交前必须编译通过 |

| **推荐** | 遵循 RAII 原则管理资源 |
| **推荐** | 使用 `auto` 关键字简化类型声明 |

## 64 位程序移植规范

| 规则 | 说明 |
|------|------|
| **强制** | 使用 `int64_t` 或 `long long` 代替 `long`（需要跨平台时） |
| **强制** | 使用 `ptrdiff_t` 处理指针运算 |
| **强制** | 使用 `size_t` 处理大小和索引 |
| **强制** | 避免将指针强制转换为 `int` |
| **推荐** | 使用 `static_assert` 进行编译时检查 |

详见 [恒生 C、C++ 编码规范 - 64 位程序移植规范](./恒生C、C++编码规范.md#八-64-位程序移植规范)
