# SQL 编码规范

## 概述

本规范为恒生电子股份有限公司 SQL 编码规范的分则，规定了 SQL 数据库对象命名、程序编写等方面的要求。

详见 [编码规范 SQL 分则](./编码规范sql分则.md)

## 对象命名规则

| 规则 | 说明 |
|------|------|
| **强制** | 对象命名应保持一致的风格，全部采用小写，中间以下划线（_）连接 |
| **强制** | 表名应有明确前缀，按规则区分不同类型 |

### 表命名规则

| 前缀 | 说明 |
|------|------|
| `t_` | 表示数据表，放置收集来的原始数据 |
| `v_` | 表示视图 |

详见 [编码规范 SQL 分则 - 1.1 表命名](./编码规范sql分则.md#11-表命名)

### 字段、变量命名规则

| 前缀 | 说明 |
|------|------|
| `sz` | 字符串 |
| `c` | 字符 |
| `i` | 整数 |
| `f` 或 `nmr` | 浮点数（双倍浮点数） |
| `d` | 日期 |
| `b` | 布尔 |
| `t` | TEXT 文本 |
| `im` | Image |

| 规则 | 说明 |
|------|------|
| **强制** | 不同表中的同名字段，应保持相同的含义 |

详见 [编码规范 SQL 分则 - 1.2 字段、变量命名](./编码规范sql分则.md#12-字段变量命名)

### 存储过程命名规则

| 规则 | 说明 |
|------|------|
| **强制** | 存储过程命名使用小写前缀 `hsp_` |
| **推荐** | 不使用 `sp` 前缀，避免与 SQL SERVER 系统本身存储过程冲突 |

详见 [编码规范 SQL 分则 - 1.3 存储过程命名](./编码规范sql分则.md#13-存储过程命名)

### 触发器命名规则

| 规则 | 说明 |
|------|------|
| **强制** | 触发器命名使用小写前缀 `htg_` |

详见 [编码规范 SQL 分则 - 1.4 触发器命名](./编码规范sql分则.md#14-触发器命名)

### 索引命名规则

| 规则 | 说明 |
|------|------|
| **强制** | 主索引命名：`idx_` + 表名 |
| **强制** | 其他索引：`idx_` + 表名 + 简要说明索引意义的串 |
| **强制** | 索引名长度不超过 30 字符 |

详见 [编码规范 SQL 分则 - 1.8 索引名](./编码规范sql分则.md#18-索引名)

## 源程序规范

### 源文件头部描述

| 规则 | 说明 |
|------|------|
| **强制** | 源文件头部描述参见《总则》说明 |

详见 [编码规范 SQL 分则 - 2.1 源文件头部描述](./编码规范sql分则.md#21-源文件头部描述)

### 过程、函数命名规则

| 规则 | 说明 |
|------|------|
| **强制** | 函数名应清晰反映函数的功能、用途 |
| **推荐** | 推荐使用动宾结构 |
| **推荐** | 名称不宜过长，一般不超过 30 字符 |

详见 [编码规范 SQL 分则 - 2.2 过程、函数命名](./编码规范sql分则.md#22-过程函数命名)

### 过程、函数头部描述规则

| 规则 | 说明 |
|------|------|
| **强制** | 对过程、函数都要有头部描述 |
| **强制** | 先写好头部描述，再编写过程或函数，即先设计再编程 |

头部描述应包含：

| 项目 | 说明 |
|------|------|
| 功能 | 中文名称，简述函数或过程的功能 |
| 输入参数 | 列出主要的、不易理解的参数含义和取值范围 |
| 输出结果（对函数） | 返回值含义 |
| 引用的外部变量说明 | (可选) |
| 关键的过程和函数 | 画出它的流程（伪代码） |
| 开发者自己的测试结果 | 如果有测试开发的文档和程序，要保留并标明保留的位置标志 |

详见 [编码规范 SQL 分则 - 2.3 过程、函数头部描述](./编码规范sql分则.md#23-过程函数头部描述)

### 语句、过程、函数体书写规范

| 规则 | 说明 |
|------|------|
| **强制** | 变量声明必须放在头部 |
| **强制** | 内容超过一行时，select、update、insert、where、and 子句一定要分行写 |
| **强制** | 将关键字提到前面醒目的位置 |
| **强制** | 注释对于多条采用 `/* */`，对于单条采用 `--` |
| **强制** | 尾部保留一行空行，以防止生成加密文件时最后的执行指令与下一文件第一行连接在一起 |
| **推荐** | 合理使用空格分隔语句中的各种元素，使源程序舒展、可读 |
| **推荐** | 赋值符前后加空格 |
| **推荐** | 使用 TAB 键进行空格时，提交时应将 TAB 全部转化为空格 |

详见 [编码规范 SQL 分则 - 2.4 语句、过程、函数体书写规范](./编码规范sql分则.md#24-语句过程函数体书写规范)

## SQL 表更改顺序

| 规则 | 说明 |
|------|------|
| **强制** | 为防止数据库发生死锁，应遵守《数据库设计说明书》中的 'SQL 表更改顺序' 章节 |
| **强制** | 项目编码人员在编写/修改数据库代码时，必须遵守更改顺序约定 |

详见 [编码规范 SQL 分则 - 3 SQL 表更改顺序](./编码规范sql分则.md#3-sql-表更改顺序)
