# 日志规范

## 概述

本规范基于公司架构办日志规范，用于指导项目日志设计与开发。

## 日志分类

### 按使用者分类

| 日志类别 | 使用者 | 日志级别 | 用途 |
|----------|--------|----------|------|
| 业务信息日志 | 业务人员 | INFO/ERROR | 定位业务异常、监控批处理进度 |
| 操作日志 | 业务人员 | INFO/ERROR | 合规稽核与操作追溯 |
| 系统运维日志 | 运维人员 | ERROR/FATAL | 排查硬件/网络/配置故障 |
| 事件日志 | 运维人员 | EVENT | 记录系统启停及关键状态变更 |
| 警告日志 | 运维人员 | WARN | 提前发现潜在风险并处理 |
| 链路日志 | 运维人员 | INFO | 性能分析与跨系统故障追踪 |
| 程序运行日志 | 开发人员 | INFO/ERROR/FATAL | 定位代码缺陷或组件异常 |
| 调试日志 | 开发人员 | DEBUG | 开发阶段功能调试与问题复现 |

### 日志级别

| 级别 | 说明 |
|------|------|
| DEBUG | 用于记录详细的调试信息，通常只在开发阶段使用 |
| INFO | 用于记录一般性的信息，一般不需要立即人工干预 |
| EVENT | 用于记录系统重要事件，例如系统启动、系统停止、配置变更 |
| WARN | 用于记录警告信息，表示系统运行中可能出现问题和风险 |
| ERROR | 用于记录错误信息，表示程序运行中出现严重问题 |
| FATAL | 用于记录致命错误信息，表示程序运行中出现极其严重的问题 |

## 错误号规范

| 错误类型 | 错误号范围 | 说明 |
|----------|------------|------|
| 程序错误 | -1 ~ -9999 | 程序缺陷导致的错误 |
| 运维错误 | 1 ~ 11000 | 运维人员需要干预的错误 |
| 业务错误 | 11000+ | 业务相关的错误 |

## 日志格式

### 统一格式

```
log_version timestamp log_level processID-Ttid systemName-moduleName-MfunName-LlineNum formatType type info ext
```

### 日志类型

| 类型 | 文件后缀 | 说明 |
|------|----------|------|
| TP0 | .debuglog | 调试日志 |
| TP1 | .tracelog | 链路日志 |
| TP2 | .bizlog | 业务信息日志 |
| TP3 | .eventlog | 事件日志 |
| TP6 | .sysopslog | 系统运维日志 |
| TP7 | .warninglog | 警告日志 |

## 记录规范

### 异常日志记录

| 场景 | 规范描述 |
|------|----------|
| 业务系统记录异常日志 | 记录异常的第一手信息，包括时间、空间、异常信息 |
| 系统结束点 | 在最外层接口调用结束后打印日志 |
| 模块内部调用 | 被调用的模块捕获异常并定义错误信息，上层不重置错误号和错误信息 |
| 异构系统调用 | 异构系统返回的异常信息需要重置，原错误信息放入扩展信息 |
| 对客接口返回 | 可以重置异常信息以保证报错信息的安全性和友好性 |

### 链路日志记录

| 场景 | 说明 |
|------|------|
| 业务系统入口和出口 | 记录业务系统的框架收到请求和开始处理业务的地方 |
| 业务系统调用其他业务系统 | 调用其他业务系统时记录链路日志 |
| 业务系统调用技术组件 | 根据条件（如返回时间限额）记录链路日志 |
| 异构系统调用 | 调用方和被调用方必须记录链路日志，不能采样 |

### 事件日志记录

| 场景 | 说明 |
|------|------|
| 系统启动/停止 | 记录系统启动、停止以及加载的组件信息 |
| 系统状态/日期变更 | 记录系统全局状态和日期变更信息 |
| 运维操作执行 | 记录运维操作执行功能，如参数设置、回退、限流、熔断等 |

## 日志规范执行

### 强制规范

| 编号 | 规范描述 |
|------|----------|
| 1000 | 系统内部错误码需要具备唯一性，每个错误码有对应的错误信息、原因说明和解决建议 |
| 2000 | 程序错误：错误号为负数；运维错误：1~11000；业务错误：11000以上 |
| 2001 | 程序错误和运维错误公司级统一，需要公司级的管控 |
| 4000 | 日志记录必须采用技术平台提供的方法 |
| 4001 | 日志记录时需要屏蔽敏感信息，敏感信息的屏蔽要做到不可逆性 |
| 4002 | 生产环境禁止默认采用DEBUG级别运行，建议使用INFO级别 |
| 4004 | 控制单条日志大小，不能超过上限（建议默认100K，不超过1M） |
| 4005 | 日志记录队列必须设置上限（建议默认1000条，不超过10万条） |
| 4006 | 循环中禁止加调试信息和链路信息 |
| 4007 | 禁止直接输出日志到控制台 |
| 5002 | 链路日志一般采用日志中心记录 |
| 5003 | 单文件不超过100M，运行目录下总日志文件不超过5G |
| 5004 | 日志数据支持保留多天日志，至少15天 |
| 5005 | 系统运维日志和程序运行日志需要长期保留（至少100天） |

### 推荐规范

| 编号 | 规范描述 |
|------|----------|
| 3002 | 重要的业务流水，要包括trace_id字段 |
| 5000 | 日志归集时命名主题按照平台标识分开 |
| 5001 | 文件保存规范：除操作日志外，其他日志均可配置保存本地文件 |

详见 [公司架构办日志规范（最新稿）](./公司架构办日志规范（最新稿）.md#规范执行)

## 日志存储

### 文件命名规范

| 日志类型 | 文件特征 |
|----------|----------|
| 业务信息日志 | biz |
| 系统运维日志 | sys |
| 程序运行日志 | dev |
| 链路日志 | trace |
| 事件日志 | event |
| 警告日志 | warn |
| 调试日志 | debug |

### 容量管理

| 规则 | 说明 |
|------|------|
| 单文件大小 | 不超过 100M |
| 总日志文件 | 不超过 5G |
| 保留天数 | 至少 15 天 |
| 系统运维/程序运行日志 | 至少保留 100 天 |
