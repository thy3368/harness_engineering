# 安全编码规范

## 概述

本规范参考 OWASP Top 10 Proactive Controls、OpenSSF Security-Focused Guide、Google、Microsoft 等大厂安全实践，为 AI 编码助手提供安全约束。

> **核心原则**：AI 生成代码并非安全无虞，必须进行安全审查；用户输入应始终被视为不可信。

详见 [OpenSSF Security Guide](https://best.openssf.org/Security-Focused-Guide-for-AI-Code-Assistant-Instructions.html)

## 通用安全原则

### 输入验证与输出编码

| 规则 | 说明 |
|------|------|
| **强制** | 所有外部输入必须验证格式、长度、类型 |
| **强制** | 使用参数化查询防止 SQL 注入 |
| **强制** | 用户生成内容渲染前必须转义 |
| **强制** | 生成 HTML/SQL 时使用安全框架或编码函数 |
| **强制** | 验证所有函数参数 |

### 认证与授权

| 规则 | 说明 |
|------|------|
| **强制** | 禁止在代码中硬编码 API Key、密码、密钥 |
| **强制** | 使用环境变量或安全 vault 引用敏感信息 |
| **强制** | 使用行业标准库处理密码和 Token |
| **强制** | 实施基于角色的访问控制（RBAC） |
| **强制** | 使用常数时间比较防止时序攻击 |

### 错误处理与日志

| 规则 | 说明 |
|------|------|
| **强制** | 错误消息不暴露内部细节（堆栈、路径、配置） |
| **强制** | 记录日志但不泄露敏感信息 |
| **强制** | 使用可配置安全的日志框架 |
| **强制** | 用户可见错误使用通用消息 |

### 安全默认值

| 规则 | 说明 |
|------|------|
| **强制** | 默认使用 HTTPS |
| **强制** | 使用强加密算法，禁用弱协议 |
| **强制** | 遵循最小权限原则 |
| **强制** | 配置使用安全默认值 |

## OWASP Top 10 防护

### A01 - 访问控制失效

| 规则 | 说明 |
|------|------|
| **强制** | 实施功能级别访问控制 |
| **强制** | 默认拒绝访问 |
| **强制** | 禁用目录遍历和元字符 |

### A02 - 身份认证失败

| 规则 | 说明 |
|------|------|
| **强制** | 使用强密码策略 |
| **强制** | 账户锁定机制防止暴力破解 |
| **强制** | 使用 BCrypt/scrypt/Argon2 哈希密码 |
| **强制** | 多因素认证 |

### A03 - 注入攻击

| 规则 | 说明 |
|------|------|
| **强制** | 使用参数化查询 |
| **强制** | 使用存储过程 |
| **强制** | 限制特殊字符输入 |
| **强制** | 使用 ORM 时注意注入 |

### A04 - 不安全设计

| 规则 | 说明 |
|------|------|
| **强制** | 安全架构设计评审 |
| **强制** | 威胁建模 |
| **强制** | 安全设计模式 |

### A05 - 安全配置错误

| 规则 | 说明 |
|------|------|
| **强制** | 安全的默认配置 |
| **强制** | 禁用调试模式 |
| **强制** | 移除默认账户和密码 |
| **强制** | 安全 Headers 配置 |

### A06 - 敏感数据泄露

| 规则 | 说明 |
|------|------|
| **强制** | 静态加密（HTTPS、TLS） |
| **强制** | 动态加密（AES-256） |
| **强制** | 敏感信息不存储明文 |
| **强制** | 密钥轮换策略 |

### A07 - 身份认证和会话管理

| 规则 | 说明 |
|------|------|
| **强制** | 安全 Cookie 配置（HttpOnly、Secure、SameSite） |
| **强制** | Session 超时 |
| **强制** | Session ID 随机生成 |
| **强制** | 登出时销毁 Session |

### A08 - 软件供应链安全

| 规则 | 说明 |
|------|------|
| **强制** | 使用官方包管理器 |
| **强制** | 锁定依赖版本 |
| **强制** | 定期更新依赖 |
| **强制** | 生成 SBOM（Software Bill of Materials） |
| **强制** | 使用 Dependency-Check 检查漏洞 |

### A09 - 完整性保护失败

| 规则 | 说明 |
|------|------|
| **强制** | 数字签名验证 |
| **强制** | 完整性校验（SHA-256） |
| **强制** | 不使用不安全的序列化 |

### A10 - 日志和监控不足

| 规则 | 说明 |
|------|------|
| **强制** | 记录安全事件 |
| **强制** | 登录失败日志 |
| **强制** | 访问日志 |
| **强制** | 监控和告警 |

## 语言特定安全规则

### C/C++

| 规则 | 说明 |
|------|------|
| **强制** | 使用边界检查函数（strncpy/strlcpy 替代 strcpy） |
| **强制** | 避免危险函数（gets、strcpy、sprintf） |
| **强制** | 启用编译器安全防御（stack canaries、DEP/NX） |
| **强制** | 包含缓冲区大小常量防止溢出 |
| **推荐** | 使用 SafeCLib 等安全库 |

### Rust

| 规则 | 说明 |
|------|------|
| **强制** | 避免 unsafe 块（除非绝对必要） |
| **强制** | 记录所有 unsafe 使用并说明理由 |
| **强制** | 运行 clippy 检查 |
| **推荐** | 使用 memsec 加密敏感数据 |

### Go

| 规则 | 说明 |
|------|------|
| **强制** | 构建时使用 data race detector |
| **强制** | 使用 crypto/subtle.ConstantTimeCompare |
| **强制** | 避免硬编码敏感信息 |
| **推荐** | 使用标准库加密包 |

### Python

| 规则 | 说明 |
|------|------|
| **强制** | 禁止 exec/eval 处理用户输入 |
| **强制** | 使用 subprocess 时 shell=False |
| **强制** | 遵循 PEP 8 |
| **强制** | 使用类型注解 |

### Java

| 规则 | 说明 |
|------|------|
| **强制** | 使用预编译语句（PreparedStatement） |
| **强制** | 使用 BCryptPasswordEncoder |
| **强制** | 避免反序列化漏洞 |
| **强制** | 使用 Spring 安全注解 |

### JavaScript/TypeScript

| 规则 | 说明 |
|------|------|
| **强制** | 避免 innerHTML，使用 textContent |
| **强制** | 使用 DOMPurify 净化 HTML |
| **强制** | Node.js 使用 prepared statements |
| **强制** | 避免 eval() |

## 平台安全规则

### Web 应用

| 规则 | 说明 |
|------|------|
| **强制** | 设置安全 Headers：Content-Security-Policy、X-Frame-Options、X-Content-Type-Options |
| **强制** | 使用 HttpOnly 和 Secure Cookie |
| **强制** | CSRF Token 验证 |
| **强制** | CORS 严格配置 |

### 云服务（AWS/Azure/GCP）

| 规则 | 说明 |
|------|------|
| **强制** | 使用云 KMS 管理密钥 |
| **强制** | 静态加密 |
| **强制** | 网络 ACL 限制 |
| **强制** | 遵循云安全最佳实践 |

### 容器

| 规则 | 说明 |
|------|------|
| **强制** | 使用最小基础镜像 |
| **强制** | 不以 root 用户运行 |
| **强制** | 使用不可变标签（SHA256） |
| **强制** | 验证镜像签名（cosign/notation） |
| **强制** | 使用官方可信来源镜像 |

### CI/CD

| 规则 | 说明 |
|------|------|
| **强制** | 密钥存储在 GitHub Secrets |
| **强制** | 不在 workflow 文件中硬编码密钥 |
| **强制** | 运行 SAST/DAST 扫描 |
| **强制** | 运行依赖检查 |
| **强制** | 锁定 action 版本到 SHA |

### IaC（Terraform/CloudFormation）

| 规则 | 说明 |
|------|------|
| **强制** | 限制资源访问 |
| **强制** | 使用安全存储密钥 |
| **强制** | 验证输入 |
| **强制** | 锁定依赖版本到 SHA |
